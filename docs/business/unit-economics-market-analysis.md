# Bulgaria Real Estate Platform — Unit Economics & Market Analysis

## 1. Executive Summary

Bulgaria is the EU's fastest-growing residential real estate market (+17–19% YoY price growth, 2025), yet its listing ecosystem is fragmented across 30+ portals, agencies, classifieds, and social channels with zero unified search experience. The platform aggregates all sources into one buyer-oriented marketplace with 3D map, AI chat, and transparent pricing — capturing the gap that exists between fragmented supply and growing demand from both domestic and EU buyers.

**Key thesis:** Fragmented distribution × growing demand × zero dominant aggregator = large capture opportunity with strong unit economics.

## 3. The Problem: Fragmented Distribution

### 3.1 Platform Landscape

Bulgaria has **30+ separate platforms** covering real estate, each with partial coverage and no unified experience:

| Category | Count | Examples | Issue |
|---|---|---|---|
| Major portals | 4 | imot.bg, Homes.bg, imoti.net, property.bg | Each has partial inventory; no single source of truth |
| Classifieds | 4 | OLX.bg, alo.bg, Bazar.bg, Imoti.info | Mixed with other categories; inconsistent data |
| Agency portals | 10+ | Address.bg, SUPRIMMO, LUXIMMO, Yavlena, Home2U, BulgarianProperties | Each agency site shows only own listings |
| International | 3 | Domaza, Indomio.bg, Realistimo | Partial overlap, multi-language duplication |
| STR platforms | 3 | Airbnb, Booking.com, Vrbo | No public API; anti-scraping; partner-only |
| Social channels | 6+ | Telegram groups, Viber, Facebook groups | Unstructured; lead-only; consent-gated |
| Official registers | 2 | Property Register, KAIS Cadastre | Verification-only; not listing sources |
| Analytics vendors | 2 | AirDNA, Airbtics | Market intelligence; not end-user facing |

**Result:** A buyer searching for an apartment in Varna must check 10–15 different websites, compare inconsistent data, and has no way to see all options on a single map.

### 3.2 Distribution Inefficiency

```
Typical buyer journey today:
  Check imot.bg → 40% of supply
  Check Homes.bg → 30% of supply (60% overlap with imot.bg)
  Check alo.bg → 20% of supply (50% overlap)
  Check 3 agency sites → 10% unique each
  Check Telegram/Viber → occasional unique deals
  Result: sees ~70% of actual supply, misses 30%
```

**Our platform aggregates 95%+ of supply into one interface**, deduplicates cross-posted listings, and presents a single unified view.

### 3.3 Ownership Group Concentration

Despite 30+ portals, **3 ownership groups** control ~60% of inventory:

| Group | Brands | Share estimate |
|---|---|---|
| Rezon Media | imot.bg, Bazar.bg | ~25% |
| Stoyanov Enterprises | SUPRIMMO, LUXIMMO, property.bg | ~15% |
| Realto Group | Address.bg, Imoteka.bg, Unique Estates | ~20% |
| Independent portals | Homes.bg, OLX.bg, alo.bg, imoti.net | ~30% |
| Long tail (agencies) | 15+ agency sites | ~10% |

This concentration means partnership deals with 3–4 groups unlock majority supply.

## 6. Competitive Positioning

### 6.1 Why No Dominant Aggregator Exists Yet

1. **Language barrier:** Bulgarian-only content limits international aggregator entry
2. **Small market perception:** EU investors overlook Bulgaria (€221M investment volume vs UK €680B)
3. **No API ecosystem:** Zero portals offer public APIs — barrier to aggregation
4. **Agency resistance:** Agencies cross-post manually rather than feed data programmatically
5. **Technical debt:** Existing portals use legacy tech (server-rendered HTML, no mobile-first)

Inspired by [LUN.ua](https://lun.ua) (Ukraine's leading real estate aggregator), adapted for Bulgaria:

| LUN.ua Feature | Our Adaptation |
|---|---|
| Aggregator across 100+ sources | 30+ Bulgarian sources (tier-1 through tier-4) |
| AI duplicate detection | Cross-source dedupe via address + price + area + photo hash |
| 3D models of complexes | OSM 3D buildings for Varna city (MapLibre + deck.gl) |
| Construction progress photos | Owner/developer upload + social monitoring |
| Map + feed split view | MapLibre 3D map (left) + scrollable feed (right) |
| Developer catalog | Developer project showcases with unit availability |
| Price analytics | Price history charts, sqm trends, yield calculators |
| Mobile-first | Responsive-first with PWA support |

### 7.2 Key Difference: Buyer-Oriented Marketplace

- **Owners post directly** — the platform is oriented on the buyer experience
- **Realtors are not mentioned** — they appear as "owner representative" managing multiple properties
- Platform is a **marketplace where owners post for users (buyers/renters)**
- Agents can manage properties on behalf of owners but the UI frames it as owner-to-buyer

### 7.3 Revenue Flywheel

```
More sources aggregated
  → More complete listings
    → More buyer traffic
      → More leads for agencies
        → Agency subscriptions increase
          → Revenue funds more source integration
            → Cycle reinforces
```

## 8. 3D Map Integration (Varna MVP)

### 8.1 Technology Stack

| Component | Technology | Purpose |
|---|---|---|
| Map renderer | MapLibre GL JS | Open-source vector map rendering |
| 3D building layer | deck.gl / MapLibre 3D extrusion | Building height visualization |
| Building data | OpenStreetMap (Varna extract) | Building footprints + height/levels |
| Data pipeline | GeoFabrik extract → osmium → tippecanoe | OSM to vector tiles |
| Tile serving | Self-hosted MBTiles or PMTiles | Zero external dependency |
| Property overlay | Custom GeoJSON layer | Pin listings on buildings |

### 8.2 OSM Data for Varna

OpenStreetMap has building footprint coverage for Varna including:
- Building outlines with `height` and `building:levels` tags
- Street network with names
- Points of interest (schools, hospitals, parks, transport)
- Coastline and water features

**Pipeline:**
1. Download Bulgaria extract from GeoFabrik (~100MB PBF)
2. Clip to Varna bounding box (43.15°N–43.25°N, 27.85°E–27.98°E)
3. Extract buildings with osmium-tool
4. Convert to GeoJSON with ogr2ogr
5. Generate MBTiles with tippecanoe (zoom 10–18)
6. Serve via PMTiles (static file, no tile server needed)

### 8.3 Blosm / Alternative Tools

[Blosm](https://github.com/vvoovv/blosm) (Blender addon) can import OSM 3D buildings for Varna:
- Useful for generating **marketing renders** (3D city views for landing page)
- Not for runtime web rendering — use MapLibre/deck.gl instead
- Can export OBJ meshes for use in Three.js if needed for hero sections

**Recommendation:** Use MapLibre GL JS 3D extrusion for the interactive map; use Blosm only for static marketing renders.
