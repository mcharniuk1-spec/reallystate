# Web Frontend Next.js

## Use When

- Implementing `/admin`, `/listings`, `/map`, `/chat`, `/settings` pages.
- Converting API contracts into React components.
- Improving frontend state and error handling.

## Workflow

1. Confirm API contract shape and empty/error states.
2. Build typed data adapters and resilient UI states.
3. Implement loading, retry, and failure boundaries.
4. Add visual hierarchy and operator-first interactions.
5. Validate with snapshots or browser checks where available.

## Component Rules

- Keep list/detail contracts explicit.
- Avoid hidden side effects in client data fetching.
- Prefer composable, testable UI blocks.

## Output

```text

Data contracts:
Components added/updated:
States covered (loading/empty/error/success):
Verification:
```
