# Web Performance & Accessibility

## Use When

- Improving dashboard/listings/map UI responsiveness.
- Fixing accessibility gaps.
- Preparing UI for stable operator use at scale.

## Checklist

1. Performance
   - reduce unnecessary re-renders
   - avoid expensive synchronous UI work
   - optimize list/table rendering
2. Accessibility
   - semantic headings and landmarks
   - keyboard navigation and focus states
   - contrast and status readability
3. Reliability
   - clear error states
   - deterministic loading behavior
   - resilient empty-state handling
